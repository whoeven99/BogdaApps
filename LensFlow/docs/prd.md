# Shopify 眼镜与隐形眼镜参数化商品、前台规格选择与订阅订单系统 PRD

- 版本：`V1.2`
- 文档类型：正式产品需求文档
- 适用对象：产品、设计、前端、后端、测试、实施、运营
- 更新说明：将产品目标从“镜片规则与诊断”升级为“参数化商品管理 + 前台规格录入匹配库存下单 + 订阅合同与周期订单闭环”，并补充镜片、隐形眼镜、自定义参数、单位规格的完整要求。

## 1. 产品概述

本项目面向 Shopify 眼镜商家，提供一套围绕“后台创建和管理商品及参数模板、前台录入消费者参数并匹配库存货品、支持订阅商品和订阅合同闭环”的完整系统。产品的核心不是单纯展示镜片选项，而是把复杂的配镜/隐形眼镜参数转成标准化商品配置，并在商品详情页完成参数采集、库存匹配、一次性订单或订阅订单创建。

## 2. 背景与问题

- 眼镜镜片和隐形眼镜商品往往不是直接按单一 SKU 销售，而是由多个参数组合决定最终可售货品。
- 消费者需要输入 `SPH`、`CYL`、`AXIS`、`ADD`、`PD`、基弧 `BC`、直径 `DIA`、含水量、包装规格等参数，普通 Shopify 商品页难以完整承载。
- 不同商家的参数名称、单位和展示方式不一致，系统必须支持标准参数，也必须允许商家自定义参数和单位。
- 同一商品下，不同参数组合对应不同库存货品、不同价格，甚至不同的订阅方案。
- 若前台没有参数校验和库存匹配能力，商家只能手工确认订单，效率低且易出错。
- 若商品支持订阅，系统还需要在首次下单时创建订阅合同，并在后续周期内生成订单和提醒，而不仅仅是展示周期方案。

## 3. 项目目标

- 提供商家后台，用于创建和管理产品、参数模板、参数值、单位规格、库存映射和订阅商品。
- 提供商品详情页主题插件，通过按钮或弹层让消费者输入商家预设参数。
- 根据消费者输入参数、当前商品、库存状态、订阅配置，自动找到对应货品并创建订单。
- 对支持订阅的商品，完成 `Selling Plan / Subscription Contract` 对接、合同创建、后续周期订单和状态提醒。
- 同时支持 `镜片` 与 `隐形眼镜` 两大类商品，并允许商家扩展自定义参数。
- 保证参数体系可配置、单位可定义、校验规则可扩展，避免把行业逻辑硬编码在前端页面中。

## 4. 非目标

- 本期不提供医生处方 OCR 识别。
- 本期不处理处方数据医学合规审核。
- 本期不覆盖线下门店 ERP 集成。
- 本期不自建完整支付系统，订单与订阅流程以 Shopify 官方能力为主。
- 本期不做复杂 BI 分析平台，仅保留基础操作记录和必要业务提醒。

## 5. 目标用户

### 5.1 消费者

在商品详情页点击“选择镜片规格”或“选择隐形眼镜参数”按钮，输入参数后匹配对应货品并完成购买或订阅。

### 5.2 商家运营

创建商品参数模板、管理参数值与单位、绑定库存货品、配置订阅商品、处理订单与合同提醒。

### 5.3 客服与实施人员

查看消费者提交的参数、匹配结果、订单状态、订阅状态，协助商家排查未命中库存或参数不完整的问题。

### 5.4 管理员与技术团队

维护参数模型、商品映射、主题插件、库存匹配逻辑、订阅合同与周期订单流程。

## 6. 核心业务场景

- 商家创建一个“镜片商品”，定义 `SPH/CYL/AXIS/ADD/PD` 等参数，并为不同参数组合绑定真实变体或货品。
- 商家创建一个“隐形眼镜商品”，定义 `SPH/BC/DIA/颜色/周期/包装片数` 等参数，并为每组参数绑定库存。
- 消费者在商品详情页点击按钮，填写或选择参数，系统实时校验单位和取值范围。
- 系统根据参数命中具体库存货品，若库存充足则生成订单。
- 若商品属于订阅商品，系统在首次下单时创建订阅合同，并在后续自动生成周期订单。
- 商家收到“合同已创建”“即将续订”“续订失败”“库存不足”等提醒。
- 消费者收到“订阅成功”“下一次发货时间”“扣款失败”或“库存异常需确认”等提醒。

## 7. 成功指标

- 参数提交后成功匹配库存货品的比例提升。
- 因参数录入错误导致的人工改单量下降。

- 镜片与隐形眼镜参数模板复用率提升。
- 订阅商品的首次合同创建成功率提升。
- 周期订单按计划生成率提升。
- 因库存不匹配导致的下单失败率下降。

## 9. 参数模型定义

### 9.1 参数体系原则

- 系统必须同时支持标准行业参数和商家自定义参数。
- 每个参数必须具备：`参数代码`、`展示名称`、`数据类型`、`单位`、`是否必填`、`校验规则`、`可选值或范围`。
- 参数单位必须可配置，且支持数值单位、文本单位、无单位三类。
- 同一个参数模板可被多个商品复用。
- 参数模板需要支持商品级覆盖，允许商家在单个商品上增删特殊参数。

### 9.2 标准镜片参数

- `prescription_type`：处方类型，如 `non_prescription`、`single_vision`、`progressive`、`reading`
- `left_sph` / `right_sph`：球镜度数，单位通常为 `D`
- `left_cyl` / `right_cyl`：散光度数，单位通常为 `D`
- `left_axis` / `right_axis`：轴位，单位通常为 `degree`
- `add_power`：附加光度，单位通常为 `D`
- `pd`：瞳距，单位通常为 `mm`
- `prism`：棱镜度数，单位通常为 `△`
- `lens_index`：折射率，如 `1.56`、`1.61`、`1.67`
- `lens_material`：材质
- `coating`：镀膜
- `tint`：染色或颜色

### 9.3 标准隐形眼镜参数

- `sph`：球镜度数，单位通常为 `D`
- `cyl`：散光度数，单位通常为 `D`
- `axis`：轴位，单位通常为 `degree`
- `bc`：基弧，单位通常为 `mm`
- `dia`：直径，单位通常为 `mm`
- `power_add`：近用附加光度，单位通常为 `D`
- `color`：颜色
- `replacement_cycle`：更换周期，如 `daily`、`biweekly`、`monthly`
- `pack_size`：包装片数，单位通常为 `pcs`
- `water_content`：含水量，单位通常为 `%`
- `brand_series`：系列

### 9.4 自定义参数与单位

- 商家必须可以自定义参数名称、参数代码、参数类型、参数单位、显示方式、取值规则和前台提示文案。
- 参数类型至少支持 `number`、`text`、`select`、`multi_select`、`boolean`。
- 单位至少支持 `D`、`mm`、`%`、`pcs`、`box`、`pair`，并允许商家新增自定义单位。

### 9.5 参数校验规则

- 必填校验
- 数值范围校验
- 步进校验，如 `SPH` 每次 `0.25D`
- 单位一致性校验
- 左右眼成对校验
- 条件校验，如选了 `toric` 才要求 `axis`
- 商品级校验，如某商品不支持 `progressive`

## 10. 详细功能需求

### 10.1 商品与参数管理后台

功能目标：让商家创建和管理产品、参数模板、参数值、单位、库存映射和订阅商品。

功能说明：

- 支持创建商品配置记录，并关联真实 Shopify 商品。
- 支持选择商品类型：`镜片`、`隐形眼镜`、`护理类订阅商品`、`其他可扩展类型`。
- 支持为商品绑定一个或多个参数模板。
- 支持管理参数顺序、默认值、必填状态、校验规则和前台展示组件。
- 支持配置参数组合与真实库存货品的映射关系。
- 支持创建订阅商品，并配置是否允许一次性购买、订阅购买或两者同时支持。

验收标准：

- 商家可独立创建一个商品参数模板并绑定到指定商品。
- 商家可配置参数单位、范围、步进和前台展示方式。
- 商家可把参数组合映射到具体货品。

### 10.2 参数模板与单位规格管理

功能目标：把镜片和隐形眼镜参数标准化，并允许商家扩展。

功能说明：

- 内置行业标准参数库。
- 支持创建自定义参数。
- 支持为参数定义单位规格和单位显示文案。
- 支持定义参数步进，如 `SPH` 每次 `0.25D`。
- 支持定义枚举值，如更换周期 `daily/monthly`。

验收标准：

- 商家可复用标准参数模板。
- 商家可在不改代码的情况下新增自定义参数。
- 同一类商品可共享模板但允许局部覆盖。

### 10.3 参数组合与库存映射

功能目标：根据消费者输入参数匹配真实可售货品。

功能说明：

- 支持按参数组合建立映射表。
- 映射结果可指向 Shopify 变体、库存项或内部货品编码。
- 支持配置价格附加、包装规格差异和库存规则。
- 支持库存不足时给出替代货品或缺货提示。

验收标准：

- 系统能根据参数命中唯一货品。
- 系统能识别没有货品或库存不足的情况。
- 参数匹配结果可用于直接创建订单。

### 10.4 Theme App Extension 商品页插件

功能目标：在商品详情页提供“选择镜片规格”或“选择隐形眼镜参数”的前台入口。

功能说明：

- 商品页展示一个按钮，点击后打开参数输入面板或弹层。
- 输入字段完全来自后台商家配置的参数模板。
- 支持镜片和隐形眼镜两类表单布局。
- 支持校验失败时实时提示。
- 支持在匹配成功后展示最终货品摘要、价格、库存和购买方式。

交互要求：

- 插件必须兼容 Shopify 商品页常见主题结构。
- 插件加载失败时不能阻断主商品页。
- 录入面板支持桌面与移动端。

验收标准：

- 商家可通过 Theme Editor 启用插件。
- 消费者可点击按钮并填写参数。
- 插件可调用 App Proxy 并返回匹配货品结果。

### 10.5 一次性订单创建

功能目标：参数匹配后直接创建普通订单购买流程。

功能说明：

- 匹配成功后把真实货品变体加入购物车。
- 同时记录消费者提交的参数摘要，便于商家履约。
- 若价格依赖参数组合，则在加入购物车前完成价格确定。

验收标准：

- 前台可基于参数匹配结果创建普通订单。
- 商家可在订单中看到参数摘要。

### 10.6 订阅商品与周期订单

功能目标：支持商家创建订阅付费商品，并在前台完成订阅购买。

功能说明：

- 商家可创建订阅商品并配置订阅方案。
- 订阅方案支持周期、折扣、发货频率、适用商品范围。
- 首次下单时，系统传递真实 `selling_plan` 并生成订阅合同。
- 系统支持后续周期订单创建。
- 系统支持商家与消费者提醒。

提醒场景：

- 订阅创建成功
- 下次扣款或发货提醒
- 续订失败
- 合同取消
- 库存不足需人工处理

验收标准：

- 商家可创建并管理订阅商品。
- 订阅商品首次下单时可生成真实订阅合同。
- 后续周期订单可按合同计划生成。

### 10.7 订单与合同跟踪

功能目标：让商家跟踪参数匹配、订单创建、合同状态和后续履约。

功能说明：

- 查看参数提交记录
- 查看命中的货品和库存状态
- 查看普通订单创建结果
- 查看订阅合同状态和续订结果
- 查看提醒发送状态

验收标准：

- 商家可看到每次参数提交对应的货品命中结果。
- 商家可查看订阅合同及后续周期订单状态。
