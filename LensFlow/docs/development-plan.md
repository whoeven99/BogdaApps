# 开发文档与执行计划

本文档基于 `docs/prd.md` 的 `V1.2` 范围定义，作为当前版本的研发执行文档，覆盖里程碑、模块拆解、接口落地顺序、测试策略与当前进度。

## 1. 开发原则

- 先建立参数模型、单位模型和库存映射模型，再做前台插件和订单闭环。
- 后台商品配置、前台参数输入、订单创建、订阅合同流程必须共用同一套参数定义。
- 标准参数与自定义参数并存，避免把镜片或隐形眼镜行业逻辑硬编码在前台脚本中。
- `Theme App Extension` 通过 `App Proxy` 获取参数模板、匹配结果和下单结果，减少对主题模板的侵入。
- 订阅购买按“两阶段”推进：
  - 阶段一：订阅商品、订阅方案、首次下单传 `selling_plan`
  - 阶段二：订阅合同、周期订单、提醒与失败补偿
- 每完成一个业务闭环，都需要同步补测试、类型检查和 Shopify 相关校验。

## 2. 目标架构

### 2.1 后台应用

- 技术栈：`Shopify App + React Router + Polaris + Prisma`
- 能力范围：
  - 商品管理
  - 参数模板管理
  - 参数单位规格管理
  - 商品参数配置
  - 参数组合与库存映射
  - 订阅商品与订阅方案管理
  - 订单与合同跟踪

### 2.2 业务层

- 商品与 Metafield 读取：`app/services/shopify-products.server.ts`
- 参数推荐与匹配基础：`src/services/prescriptionRecommendations.ts`
- 前台聚合：`app/lib/storefront-lens.server.ts`
- 订阅方案与 Selling Plan：`app/services/subscription-plans.server.ts`
- 订阅诊断与状态汇总：`app/services/subscription-diagnostics.server.ts`
- 后续新增：
  - 参数模板服务
  - 单位规格服务
  - 参数组合映射服务
  - 订单创建服务
  - 订阅合同服务

### 2.3 前台接入

- 主题扩展：`extensions/lens-product-widget`
- 数据出口：
  - `/apps/product-match`
  - `/apps/create-parameterized-order`
  - `/apps/create-subscription-contract`
- 前台 JS：
  - 渲染商品页按钮与参数面板
  - 按参数模板动态生成字段
  - 调用匹配接口
  - 调用普通订单和订阅下单接口

- 配镜推荐页与推荐接口雏形

- 还没有正式的参数模板模型与后台管理页

- 当前前台推荐器仍是演示型结果页，不是正式订单入口
